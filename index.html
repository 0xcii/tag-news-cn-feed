    新的BRC-20S，能开启比特币上的「DeFi之夏」吗？
    BRC-20S（BRC-20 Staking）是 BRC-20 协议的拓展版本，由 OKX 提出，旨在为比特币网络引入一种针对 BRC-20 Token 或比特币的质押机制。 通过 BRC-20S，用户能够抵押自己的 BRC-20 Token 或比特币，并获得相应的 BRC-20S Token 作为回报。与 BRC-20 一样，质押奖励的计算也会由一个索引来实现。OKX 将提供一个开源接口用于利润计算，任何用户的质押收益都会在后端和开源工具中一致地计算，确保所有计算的统一性。 BRC-20S 交互流程 目前，BRC-20S 的铺开还需要等待 OKX 的索引及 BRC-20S Token 交易市场等相关基建的完善。但我们可以预想一下，BRC-20S 可能为市场带来哪些新的变化呢？ 新的 BRC Token 发行方式 目前，BRC-20 Token 的发行已经逐渐从原始的公布 Ticker（Token 名称）由大家自行刻录演变成了通过成熟的 BRC-20 Token 发行工具 Luminex 来完成。通过 Luminex 完成的原理是项目方在部署 BRC-20 Token 的时候将单张铭文可刻录的 Token 数量设置为与总量相同，先将所有 Token 刻录到自己手中，然后用户通过 Luminex 支付 Gas，最后再完成 Token 的发放。有了 Luminex 之后，白名单制度已经能够很好地运用到 BRC-20 Token 的发行中，对项目的冷启动提供了帮助。 而 BRC-20S 对直接质押比特币来「挖」BRC-20S Token 收益的支持，很可能再一次引发 BRC Token 发行模式的新变化，之后的新项目有可能直接通过质押比特币来完成初始的 Token 分发，「比特币质押挖矿」的热潮可能会被掀起。同时，BRC-20S 也支持质押 BRC-20 Token 来「挖」BRC-20S Token 收益，这将使 BRC-20 Token 的玩法得到拓展，$ORDI 这样的 BRC-20 Token 龙头会不会变成「金铲子」呢？ 「母 Token」与「子 Token」 在以太坊生态中，「双 Token 模型」已经屡见不鲜。而 BRC-20S 在技术上为比特币网络上实现「双 Token 模型」扫清了障碍，接下来的问题就是，我们可以怎么玩？ 对此我设想了一个以「双 Token 模型」的适配场景。最近的新概念「递归铭文」本质上是比特币链上铭文资源间的互相引用。如果有一个生成艺术的工作室出现，用一个 AI 模型以每日链上的所有现存铭文为素材训练出一幅生成艺术作品并拍卖，那么 BRC-20 Token 就可以作为「母 Token」发挥治理效用，比如决定每期艺术作品生成的主题（保留或去掉哪些铭文作为训练素材等）、决定每期上架的艺术作品到底是哪幅、工作室的外部合作决策等等。同时，质押这个「母 Token」得到的「子 Token」收益可以用作 Claim 这幅艺术作品的支付 Token 来发挥效用。 「双 Token 模型」的适配场景还有很多，比如 GameFi 、SocialFi 等等。结合「递归铭文」，比特币网络上想必会有更多好玩的应用出现。 展望：小图片的质押 既然已经有 BRC-20S 实现了对 BRC-20 Token 的质押，那么实现直接对小图片的质押有没有可能呢？ 比如，在协议中增加两个关键词，一个指定 NFT 系列的名称，另一个指定该系列的有效铭文序号合集（如 Inscription #10000、Inscription #10001...）。通过链下索引来确认用户所刻录的 Deposit 铭文是否有效，对有效 Deposit 铭文的持有地址进行质押奖励计算，这样就实现了小图片的质押。 结语 BRC-20S 对比特币网络上质押功能的支持，拓宽了 BRC-20 Token 的想象空间，有望对比特币及 BRC-20 Token 的流动性带来更好的正面作用。一直在发展的 Ordinals 生态，你很难不爱。
